# WorldTale 要件定義書

## 1. コンセプト

**世界はたくさんの物語でできている**

目標は世界を物語で埋め尽くすこと。ユーザーは自身の体験を日記のように書き起こし、
それが「場所 × 時」に紐づけられて蓄積されていく。誰かの過ごした時間と場所を、
別の誰かが辿って読むことができるソーシャルサイト。

- まずは日本国内を対象とする
- 仕様としてはブログサイトに近い
- プロトタイプでは粒度を **都道府県 × 年（西暦1年単位）** とする
  （将来は市区町村 × 年・季節へ細分化する想定）

## 2. 用語

| 用語 | 意味 |
|---|---|
| 物語 (Story) | ユーザーが書く体験談・日記。都道府県と年に紐づく |
| 主人公 (Author) | 物語の投稿者。自分の物語のリアクション数・閲覧数を確認できる |
| 風景写真 (Photo) | 物語に添える、その地域・その季節の風景写真 |
| 出会ってたかも | 「同じ時期に同じ場所にいたかもしれない」ことを伝えるリアクション |

## 3. ユーザーアカウント

- ユーザーが持つのは次の3つ**のみ**。メールアドレス等は一切使わない。
  1. **ユーザー名**（公開される表示名。一意でなくてよい）
  2. **ユーザーID**（登録時にランダムで割り当てられる。一意。ログインに使用）
  3. **パスワード**（ユーザー自身が設定）
- 登録時にユーザーIDを画面に表示し、「このIDは再発行できないので必ず控えてください」と明示する。
- **ユーザーID（ログインID）は認証情報であり、本人以外には一切見せない**。
  外部に公開される識別子は、登録時に自動発行される**公開ハンドル**（ランダム・一意）とし、
  ユーザーページのURL（/u/:handle）や投稿者リンクにはハンドルのみを使う。
- パスワードリセット手段は持たない（メール不使用のため）。プロトタイプでは割り切る。

### 認証が必要な操作 / 不要な操作

| 操作 | ログイン |
|---|---|
| 物語・写真の閲覧、検索、マップ閲覧 | 不要 |
| リアクション（いいね / 出会ってたかも） | **不要** |
| 物語の投稿・編集・削除 | 必要 |
| 写真のアップロード・削除 | 必要 |
| 報告（通報） | **必要** |
| 自分の物語の統計（閲覧数・リアクション数）確認 | 必要（本人のみ） |

## 4. 機能要件

### 4.1 物語の投稿

- タイトル + 本文（プレーンテキスト、改行可）を投稿できる
- 必ず **都道府県（47択）**・**市区町村（テキスト入力）**・**年（西暦）**・**季節（春/夏/秋/冬）**
  を指定する（プロトタイプでも必須）
  - 市区町村は「市区町村まで」。番地・建物名などの詳細な住所は書かせない（注意書きで明示）
  - ただし**検索・マップの分類はプロトタイプでは従来どおり都道府県 × 年**。
    市区町村と季節は分類には使わず、物語を読むときに表示される情報とする
- 物語ページ・読書モードでは 都道府県・市区町村・年・季節 が表示される
- **1ユーザーあたり、1場所（都道府県）× 1季節につき物語は1つまで**（写真と同じルール）。
  すでにある場所×季節に投稿・変更しようとした場合はエラーで案内する
- 投稿フォームに**風景写真の添付欄（任意）**を設ける（4.4 の制約に従う）
- 投稿前に注意事項を表示し、同意チェックを必須とする:
  - **個人を特定できる情報（学校名・職場名・住所・実名など）を書かない**
  - 他者を害する内容を書かない
- 投稿者は自分の物語を編集・削除できる

### 4.2 物語を読む・探す

- **マップ**: トップページに**実際の日本列島の形をしたドット絵マップ**
  （道画面と同じピクセルアート調、海と列島）を表示。
  都道府県をタップするとその場所のページへ。物語の多さで土地の色の濃淡をつける。
  将来はマップをズームイン→市区町村選択までできるようにする（プロトタイプでは都道府県まで）。
- **検索**: 「自分が過ごした時期・場所」で検索できる（都道府県 × 年の組み合わせ）。
- **ユーザーページ**: あるユーザーの物語を時系列で辿れる。
  途中で引っ越した先や旅行先での経験も、同じユーザーに紐づいて追って読める。

### 4.3 リアクション

- 読者は **「いいね」** と **「出会ってたかも」** の2種類のボタンのみでリアクションできる
- ユーザー登録は不要。匿名トークン（ブラウザごとに生成・保存）で重複を防ぐ
- 同じボタンをもう一度押すと取り消しになる（トグル）
- 主人公（投稿者）は自分の物語ごとに「いいね数」「出会ってたかも数」「閲覧数」を確認できる
- 閲覧数は物語の本文ページが開かれた回数

### 4.4 風景写真

- 物語に添える形で、その地域・その季節の風景写真をアップロードできる
- **顔が写っている写真はNG**（投稿時の同意チェック + 報告機能で運用）
- 制限: **1ユーザーあたり、1場所（都道府県）× 1季節（春/夏/秋/冬）につき1枚まで**
- アップロードされた写真は、その場所・その季節のページの**背景**となり、
  複数枚あるときはスライドショーのように順番に表示される
- 投稿者は自分の写真を削除できる（削除すれば同じ枠に再アップロード可能）

### 4.5 報告（通報）

- 物語・写真それぞれに報告ボタンを設置
- 報告にはログインが必要
- 報告理由（個人情報を含む / 顔が写っている / 不快・有害 / その他+自由記述）を選んで送信
- 報告されたコンテンツは管理側で非表示化できる（DB上の hidden フラグ）

### 4.6 SNS共有

共有ボタンは「作者用」と「読者用」を分ける:

**作者用（マイページの自分の物語それぞれに設置）**
1. **「私の物語」共有**: 自分がこの物語の主人公であることを公表する共有。
   共有文にその物語へのリンクを含み、受け取った人はその物語（とユーザー）にジャンプできる。
2. **匿名共有（そっと共有）**: 「世界のどこかに私の物語を追加しました」という文と、
   **サイトトップへのリンクのみ**を含む共有。どの物語かは分からない。

**読者用（閲覧ページ = 物語ページ・読書モードに設置）**
3. **「共有」**: 「**〜県でこんな物語を見つけました。**」という文と、
   その物語へのリンクを含む共有。読んだ物語を人にすすめるためのもの。

共有先は X (Twitter) の intent リンク + Web Share API（対応端末）とする。

## 5. 非機能要件

- **コスト**: すべて無料枠の範囲で運用する
  - Cloudflare Pages（フロントエンド）
  - Cloudflare Workers（API、無料枠 100,000 req/日）
  - Supabase Free（Postgres 500MB、Storage 1GB）
  - GitHub Student Developer Pack（ドメイン等に活用可）
- **デザイン**: レトロRPG風のゲーム調（ドット絵・メッセージウィンドウUI）。
  トーンは穏やかに保つ（どぎつい原色は避け、牧歌的なフィールドの色合い）
- **言語**: 日本語 UI（プロトタイプ）
- **写真**: 1枚あたり 5MB まで、JPEG/PNG/WebP

## 6. プロトタイプで割り切ること（将来課題）

- 市区町村・季節単位の物語紐づけ（現状は都道府県 × 年）
- パスワードリセット
- 顔検出などの自動モデレーション（現状は同意チェック + 報告 + 手動非表示）
- 多言語化・世界地図への拡張
- 管理者用モデレーション画面（現状は Supabase ダッシュボードで hidden フラグを操作）
